练习 1：实施行级安全性

在本练习中，您将强制执行行级安全性，以确保销售人员只能看到在其指定区域进行的销售。

任务 1：开始

在此任务中，您将为实验室设置环境。

重要提示：如果您从上一个实验继续（并且您成功完成了该实验），请不要完成此任务；相反，从下一个任务继续。

  1. 若要打开 Power BI Desktop，请在任务栏上单击 Microsoft Power BI Desktop 快捷方式。

  2. 要关闭入门窗口，请单击窗口左上角的X。

  3. 若要打开初始 Power BI Desktop 文件，请单击“文件”功能区选项卡以打开后台视图。

  4. 选择打开报告。

  5. 单击浏览报告。

  6. 在打开窗口中，导航到D:\PL300\Labs\10-row-level-security\Starter文件夹。

  7. 选择销售分析文件。

  8. 单击打开。

  9. 关闭所有可能打开的信息窗口。

  10. 要创建文件的副本，请单击“文件”功能区选项卡以打开后台视图。

  11. 选择另存为。

  12. 如果提示应用更改，请单击应用。

  13. 在另存为窗口中，导航到D:\PL300\MySolution文件夹。

  14. 单击保存。

任务 2：实施行级安全性

在此任务中，您将强制执行行级安全性，以确保销售人员只能看到在其指定区域进行的销售。

  1. 切换到数据视图。

  2. 在字段窗格中，选择销售人员（绩效）表。

  3. 查看数据，注意到 Michael Blythe (EmployeeKey 281) 的 UPN 值为：michael-blythe@adventureworks.com

    回想一下，Michael Blythe 被分配到三个销售区域：美国东北部、美国中部和美国东南部。

  4. 切换到报告视图。

  5. 在“建模”功能区选项卡上，从“安全”组中，单击“管理角色” 。

  6. 在“管理角色”窗口中，单击“创建” 。

  7. 在框中，将所选文本替换为角色名称：Salespeople，然后按Enter。

  8. 要分配过滤器，对于销售员（绩效）表，单击省略号 (...) 字符，然后选择添加过滤器 | [UPN]。

  9. 在Table Filter DAX Expression框中，通过将“Value”替换为USERPRINCIPALNAME()来修改表达式。

    USERPRINCIPALNAME() 是一个数据分析表达式 (DAX) 函数，它返回经过身份验证的用户的名称。这意味着Salesperson (Performance)表将按查询模型的用户的用户主体名称 (UPN) 进行过滤。

  10. 单击保存。

  11. 要测试安全角色，请在“建模”功能区选项卡上的“安全”组内单击“查看为”。

  12. 在View as Roles窗口中，选中Other User项，然后在相应的框中输入：michael-blythe@adventureworks.com

  13. 检查销售人员角色。

    此配置会导致使用Salespeople角色并使用您的 Michael Blythe 的名字来模拟用户。

  14. 单击确定。

  15. 注意报告页面上方的黄色横幅，描述了测试安全上下文。

  16. 请注意，在表格视觉对象中，仅列出了销售人员Michael Blythe。

  17. 要停止测试，请在黄色横幅的右侧单击停止查看。

    将 Power BI Desktop 文件发布到 Power BI 服务时，你需要完成发布后任务以将安全主体映射到销售人员角色。你不会在这个实验室里这样做。

  18. 要删除角色，请在“建模”功能区选项卡上的“安全”组内，单击“管理角色” 。

  19. 在“管理角色”窗口中，单击“删除” 。

  20. 当提示确认删除时，单击是，删除。

  21. 单击保存。

任务 3：完成

在此任务中，您将完成实验。

  1. 保存 Power BI Desktop 文件。
