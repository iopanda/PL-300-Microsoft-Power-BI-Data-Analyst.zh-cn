在 Power BI Desktop 中准备数据

完成实验的预计时间为 45 分钟

在本实验室中，您将开始为 Adventure Works 公司开发 Power BI Desktop 解决方案。它涉及连接到源数据、预览数据以及使用数据预览技术来了解源数据的特征和质量。

在本实验中，您将学习如何：

  • 打开 Power BI 桌面

  • 设置 Power BI Desktop 选项

  • 连接到源数据

  • 预览源数据

  • 使用数据预览技术更好地理解数据

实验室故事

该实验室是一系列实验室中的众多实验室之一，这些实验室被设计成一个完整的故事，从数据准备到发布为报告和仪表板。您可以按任何顺序完成实验。但是，如果您打算通过多个实验室工作，我们建议您按以下顺序进行：

  1. 在 Power BI Desktop 中准备数据

  2. 在 Power BI Desktop 中加载数据

  3. Power BI Desktop 中的模型数据

  4. 在 Power BI Desktop 中创建 DAX 计算，第 1 部分

  5. 在 Power BI Desktop 中创建 DAX 计算，第 2 部分

  6. 在 Power BI Desktop 中设计报表，第 1 部分

  7. 在 Power BI Desktop 中设计报表，第 2 部分

  8. 创建 Power BI 仪表板

  9. 在 Power BI Desktop 中执行数据分析

  10. 实施行级安全

练习 1：准备数据

在本练习中，您将创建八个 Power BI Desktop 查询。六个查询将从 SQL Server 获取数据，两个来自 CSV 文件。

任务 1：保存 Power BI Desktop 文件

在此任务中，您将首先保存 Power BI Desktop 文件。

  1. 若要打开 Power BI Desktop，请在任务栏上单击 Microsoft Power BI Desktop 快捷方式。
  1. 要关闭入门窗口，请单击窗口右上角的X。

  1. 要保存文件，请单击“文件”功能区选项卡以打开后台视图。

  2. 选择保存。

  1. 在另存为窗口中，导航到D:\PL300\MySolution文件夹。

  2. 在文件名框中，输入Sales Analysis。

  1. 单击保存。

    提示：您也可以通过单击左上角的保存图标来保存文件。

任务 2：设置 Power BI Desktop 选项

在此任务中，您将设置 Power BI Desktop 选项。

  1. 在 Power BI Desktop 中，单击“文件”功能区选项卡以打开后台视图。

  2. 在左侧，选择选项和设置，然后选择选项。

  1. 在选项窗口左侧的当前文件组中，选择数据加载。

    当前文件的数据加载设置允许设置确定建模时默认行为的选项。

  2. 在“关系”组中，取消选中已选中的两个选项。

    虽然在开发数据模型时启用这两个选项会很有帮助，但您之前禁用了它们以支持实验室体验。在 Power BI Desktop中的加载数据实验室中创建关系时，你将了解为什么要添加每个关系。

  3. 单击确定。

  4. 保存 Power BI Desktop 文件。

任务 3：从 SQL Server 获取数据

在此任务中，您将创建基于 SQL Server 表的查询。

  1. 在主页功能区选项卡上，从数据组内单击SQL Server。

  2. 在SQL Server 数据库窗口的服务器框中，输入localhost。

    在本实验中，您将使用localhost连接到 SQL Server 数据库。在创建您自己的解决方案时，这不是推荐的做法。这是因为网关数据源无法解析localhost。

  3. 单击确定。

  4. 如果提示输入凭据，请在SQL Server 数据库窗口中选择使用我当前的凭据。然后连接。

  5. 在左侧的导航器窗口中，展开AdventureWorksDW2020数据库。

    AdventureWorksDW2020数据库基于AdventureWorksDW2017示例数据库。它已被修改以支持课程实验室的学习目标。

  6. 选择（但不要检查）DimEmployee表。

  7. 在右窗格中，请注意表数据的预览。

    预览数据允许您确定列和行样本。

  8. 要创建查询，请选中以下六个表旁边的复选框：

    • 昏暗的员工

    • DimEmployeeSalesTerritory

    • 昏暗产品

    • 昏暗经销商

    • DimSalesTerritory

    • FactResellerSales

  9. 要对选定表的数据应用转换，请单击转换数据。

    您不会在此实验室中转换数据。本实验的目标侧重于探索和分析Power Query 编辑器窗口中的数据。

任务 4：预览 SQL Server 查询

在此任务中，您将预览 SQL Server 查询的数据。首先，您将了解有关数据的相关信息。您还将使用列质量、列分布和列配置文件工具来了解数据并评估数据质量。

  1. 在Power Query 编辑器窗口的左侧，注意查询窗格。

    “查询”窗格包含您检查的每个表的一个查询。

  2. 选择第一个查询 - DimEmployee。

    SQL Server 数据库中的DimEmployee表为每个员工存储一行。此表中的一部分行表示销售人员，这将与您将开发的模型相关。

  3. 在左下角的状态栏中，请注意表统计信息——该表有 33 列和 296 行。

  4. 在数据预览窗格中，水平滚动以查看所有列。

  5. 请注意，最后五列包含表或值链接。

    这五列表示与数据库中其他表的关系。它们可用于将表连接在一起。你将在 Power BI Desktop 中的加载数据实验室中连接表。

  6. 要评估列质量，请在“查看”功能区选项卡上的“数据预览”组内，选中“列质量” 。

    列质量功能允许您轻松确定在列中找到的有效值、错误值或空值的百分比。

  7. 对于Position列（倒数第六列），请注意 94% 的行是空的（null）。

  8. 要评估列分布，请在“视图”功能区选项卡上的“数据预览”组内，选中“列分布” 。

  9. 再次查看Position列，并注意有四个不同的值和一个唯一值。

  10. 查看EmployeeKey（第一）列的列分布——有 296 个不同的值和 296 个唯一值。

    当不同计数和唯一计数相同时，表示该列包含唯一值。建模时，某些模型表具有唯一列很重要。这些独特的列可用于创建一对多关系，你将在 Power BI Desktop 中的模型数据实验室中执行此操作。

  11. 在查询窗格中，选择DimEmployeeSalesTerritory查询。

    DimEmployeeSalesTerritory表为每个员工及其管理的销售区域存储一行。该表支持将多个区域与单个员工相关联。一些员工管理一个、两个或可能更多的区域。当您对此数据建模时，您需要定义多对多关系。

  12. 在“查询”窗格中，选择DimProduct查询。

    DimProduct表包含公司销售的每种产品一行。

  13. 水平滚动以显示最后一列。

  14. 注意DimProductSubcategory列。

    在 Power BI Desktop中的加载数据实验室中向此查询添加转换时，将使用DimProductSubcategory列连接表。

  15. 在“查询”窗格中，选择DimReseller查询。

    DimReseller表包含每个经销商一行。经销商销售、分销或增值 Adventure Works 产品。

  16. 要查看列值，请在“查看”功能区选项卡上的“数据预览”组内，选中“列配置文件” 。

  17. 选择BusinessType列标题。

  18. 请注意数据预览窗格下方的新窗格。

  19. 在数据预览窗格中查看列统计信息和值分布。

  20. 注意数据质量问题：仓库有两个标签（Warehouse和拼写错误的Ware House）。

  21. 将光标悬停在Ware House栏上，注意有五行具有此值。

    你将在 Power BI Desktop 中加载数据实验室中应用转换来重新标记这五行。

  22. 在“查询”窗格中，选择DimSalesTerritory查询。

    DimSalesTerritory表包含每个销售区域的一行，包括Corporate HQ（总部）。地区被分配给一个国家，国家被分配给组。在Power BI Desktop 中的模型数据实验室中，你将创建一个层次结构以支持区域、国家或组级别的分析。

  23. 在查询窗格中，选择FactResellerSales查询。

    FactResellerSales表的每个销售订单行包含一行 — 销售订单包含一个或多个行项目。

  24. 查看TotalProductCost列的列质量，并注意 8% 的行是空的。

    缺少TotalProductCost列值是数据质量问题。为了解决此问题，在Power BI Desktop 中加载数据实验室中，你将应用转换以使用存储在相关DimProduct表中的产品标准成本来填充缺失值。

任务 5：从 CSV 文件中获取数据

在此任务中，您将创建一个基于 CSV 文件的查询。

  1. 要添加新查询，请在Power Query 编辑器窗口的主页功能区选项卡上，从新查询组内单击新源向下箭头，然后选择Text/CSV。

  2. 在打开窗口中，导航到D:\PL300\Resources文件夹，然后选择ResellerSalesTargets.csv文件。

  3. 单击打开。

  4. 在ResellerSalesTargets.csv窗口中，查看预览数据。

  5. 单击确定。

  6. 在“查询”窗格中，请注意添加了ResellerSalesTargets查询。

    ResellerSalesTargets CSV 文件包含每个销售人员每年一行。每行记录 12 个月度销售目标（以千计）。请注意，Adventure Works 公司的营业年度从 7 月 1 日开始。

  7. 请注意，没有一列包含空值。

    当没有每月销售目标时，将存储一个连字符。

  8. 查看每个列标题中列名称左侧的图标。

    图标代表列数据类型。123是整数，ABC是文本。

    在 Power BI Desktop中的加载数据实验室中，您将应用许多转换来获得仅包含三列的不同形状的结果：Date、EmployeeKey和TargetAmount。

任务 6：从 CSV 文件中获取其他数据

在此任务中，您将创建一个基于不同 CSV 文件的附加查询。

  1. 使用上一个任务中的步骤基于D:\PL300\Resources\ColorFormats.csv文件创建查询。

    ColorFormats CSV 文件包含每种产品颜色的一行。每行记录十六进制代码以格式化背景和字体颜色。你将在 Power BI Desktop 中加载数据实验室中将此数据与DimProduct查询数据集成。

任务 7：完成

在此任务中，您将完成实验。

  1. 在视图功能区选项卡上，从数据预览组中，取消选中之前在此实验室中启用的三个数据预览选项：

    • 色谱柱质量

    • 列分布

    • 柱型材

  2. 若要保存 Power BI Desktop 文件，请在Power Query 编辑器窗口中的文件后台视图上，选择保存。

  3. 当提示应用查询时，单击Apply Later。

    应用查询会将它们的数据加载到数据模型中。您还没有准备好这样做，因为必须首先应用许多转换。

  4. 如果你打算开始下一个实验室，请将 Power BI Desktop 保持打开状态。

    你将对查询应用各种转换，然后应用查询以将它们加载到 Power BI Desktop 中的加载数据实验室中的数据模型。
