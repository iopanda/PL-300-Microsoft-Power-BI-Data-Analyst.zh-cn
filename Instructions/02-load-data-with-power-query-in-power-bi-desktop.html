在 Power BI Desktop 中加载数据

完成实验的预计时间为 45 分钟

在本实验中，您将开始对上一个实验中创建的每个查询应用转换。然后，您将应用查询以将每个查询作为表加载到数据模型中。

在本实验中，您将学习如何：

  • 应用各种转换

  • 应用查询以将它们加载到数据模型中

实验室故事

该实验室是一系列实验室中的众多实验室之一，这些实验室被设计成一个完整的故事，从数据准备到发布为报告和仪表板。您可以按任何顺序完成实验。但是，如果您打算通过多个实验室工作，我们建议您按以下顺序进行：

  1. 在 Power BI Desktop 中准备数据

  2. 在 Power BI Desktop 中加载数据

  3. Power BI Desktop 中的模型数据

  4. 在 Power BI Desktop 中创建 DAX 计算，第 1 部分

  5. 在 Power BI Desktop 中创建 DAX 计算，第 2 部分

  6. 在 Power BI Desktop 中设计报表，第 1 部分

  7. 在 Power BI Desktop 中设计报表，第 2 部分

  8. 创建 Power BI 仪表板

  9. 在 Power BI Desktop 中执行数据分析

  10. 实施行级安全

练习 1：加载数据

在本练习中，您将对前一个实验室中创建的每个查询应用转换。

任务 1：开始

在此任务中，您将为实验室设置环境。

重要提示：如果您从上一个实验继续（并且您成功完成了该实验），请不要完成此任务；相反，从下一个任务继续。

  1. 若要打开 Power BI Desktop，请在任务栏上单击 Microsoft Power BI Desktop 快捷方式。

  2. 要关闭入门窗口，请单击窗口左上角的X。

  3. 若要打开初始 Power BI Desktop 文件，请单击“文件”功能区选项卡以打开后台视图。

  4. 选择打开报告。

  5. 单击浏览报告。

  6. 在打开窗口中，导航到D:\PL300\Labs\02-load-data-with-power-query-in-power-bi-desktop\Starter文件夹。

  7. 选择销售分析文件。

  8. 单击打开。

  9. 关闭所有可能打开的信息窗口。

  10. 请注意功能区下方的黄色警告消息。

    该消息提醒您查询尚未应用于加载为模型表这一事实。您将在本实验室稍后应用这些查询。

  11. 要关闭警告消息，请单击黄色警告消息右侧的X。

  12. 要创建文件的副本，请单击“文件”功能区选项卡以打开后台视图。

  13. 选择另存为。

  14. 如果提示应用更改，请单击以后应用。

  15. 在另存为窗口中，导航到D:\PL300\MySolution文件夹。

  16. 单击保存。

  17. 若要打开Power Query 编辑器窗口，请在“主页”功能区选项卡上的“查询”组内，单击“转换数据”图标。

任务 2：配置销售人员查询

在此任务中，您将配置Salesperson查询。

  1. 在Power Query 编辑器窗口的“查询”窗格中，选择DimEmployee查询。

  2. 要重命名查询，请在“查询设置”窗格（位于右侧）的“名称”框中，将文本替换为Salesperson，然后按Enter。

    查询名称将确定模型表名称。建议定义简洁而友好的名称。

  3. 在“查询”窗格中，验证查询名称是否已更新。

    您现在将过滤查询行以仅检索作为销售人员的员工。

  4. 要查找特定列，请在“主页”功能区选项卡上，单击“管理列”向下箭头，单击“选择列”向下箭头，然后选择“转到列”。

    提示：当查询包含许多列时，此技术很有用。如果没有太多列，您可以简单地水平滚动以找到感兴趣的列。

  5. 在Go to Column窗口中，要按列名称对列表进行排序，请单击AZ排序按钮，然后选择Name。

  6. 选择SalesPersonFlag列，然后单击OK。

  7. 要筛选查询，请在SalesPersonFlag列标题中单击向下箭头，然后取消选中FALSE。

  8. 单击确定。

  9. 在Query Settings窗格的Applied Steps列表中，注意添加了Filtered Rows步骤。

    您创建的每个转换都会产生额外的步骤逻辑。可以编辑或删除步骤。也可以在查询转换的那个阶段选择一个步骤来预览查询结果。

  10. 要删除列，请在“主页”功能区选项卡上，单击“管理列”组，然后单击“选择列”图标。

  11. 在“选择列”窗口中，要取消选中所有列，请取消选中（选择所有列）项。

  12. 要包括列，请检查以下六列：

    • 员工密钥

    • 雇员国民身份证备用钥匙

    • 名

    • 姓

    • 标题

    • 电子邮件地址

  13. 单击确定。

  14. 在Applied Steps列表中，请注意添加了另一个查询步骤。

  15. 要创建单个名称列，首先选择FirstName列标题。

  16. 在按住Ctrl键的同时，选择LastName列。

  17. 右键单击任一选择列标题，然后在上下文菜单中选择Merge Columns。

    通过右键单击列标题，然后从上下文菜单中选择它们，可以应用许多常见的转换。但是请注意，功能区中提供了更多转换。

  18. 在“合并列”窗口的“分隔符”下拉列表中，选择“空格”。

  19. 在New Column Name框中，将文本替换为Salesperson。

  20. 单击确定。

  21. 要重命名EmployeeNationalIDAlternateKey列，请双击EmployeeNationalIDAlternateKey列标题。

  22. 将文本替换为EmployeeID，然后按Enter。

    重要提示：当指示重命名列时，请务必按照说明重命名它们。

  23. 使用前面的步骤将EmailAddress列重命名为UPN。

    UPN 是用户主体名称的首字母缩写词。

  24. 在左下角的状态栏中，验证查询是否有 5 列和 18 行。

    重要提示：如果您的查询没有产生正确的结果，请不要继续进行，这一点很重要 - 将无法完成以后的实验。如果查询列或行不匹配，请返回此任务中的步骤以解决任何问题。

任务 4：配置产品查询

在此任务中，您将配置产品查询。

重要提示：当已提供详细说明时，实验室步骤现在将提供更简洁的说明。如果您需要详细说明，可以参考之前任务的步骤。

  1. 选择DimProduct查询。

  2. 将查询重命名为Product。

  3. 找到FinishedGoodsFlag列，然后过滤该列以检索成品产​​品（即 TRUE）。

  4. 删除所有列，以下除外：

    • 产品密钥

    • 英文产品名称

    • 标准成本

    • 颜色

    • 昏暗产品子类别

  5. 请注意，DimProductSubcategory列表示一个相关表（它包含值链接）。

  6. 在DimProductSubcategory列标题中，在列名称的右侧，单击展开按钮。

  7. 要取消选中所有列，请取消选中（选择所有列）项。

  13. 重命名以下四列：

任务 10：更新产品查询

在此任务中，您将通过合并ColorFormats查询来更新Product查询。

  1. 选择产品查询。

  2. 要合并ColorFormats查询，请在Home功能区选项卡上，单击Combine向下箭头，然后单击Merge Queries。

    合并查询允许集成数据，在这种情况下来自不同的数据源（SQL Server 和 CSV 文件）。

  3. 在“合并”窗口的“产品”查询网格中，选择“颜色”列标题。

  4. 在Product查询网格下方的下拉列表中，选择ColorFormats查询。

  5. 在ColorFormats查询网格中，选择Color列标题。

  6. 当Privacy Levels窗口打开时，对于两个数据源中的每一个，在相应的下拉列表中选择Organizational。

    可以为数据源配置隐私级别，以确定数据是否可以在源之间共享。如有必要，将每个数据源设置为组织允许他们共享数据。请注意，私有数据源永远不能与其他数据源共享。这并不意味着私有数据不能共享；这意味着 Power Query 引擎无法在源之间共享数据。

  7. 单击保存。

  8. 在Merge窗口中，使用默认的Join Kind - 保持 Left Outer 的选择，然后单击OK。

  9. 展开ColorFormats列以包括以下两列：

    • 背景颜色格式

    • 字体颜色格式

  10. 在状态栏中，验证查询现在是否有 8 列和 397 行。

任务 11：更新 ColorFormats 查询

在此任务中，您将更新ColorFormats以禁用其加载。

  1. 选择ColorFormats查询。

  2. 在查询设置窗格中，单击所有属性链接。

    禁用加载意味着它不会作为表加载到数据模型中。这样做是因为查询与Product查询合并，后者可以加载到数据模型。

  4. 单击确定。

  4. 保存 Power BI Desktop 文件。
